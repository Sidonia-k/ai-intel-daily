# 阶段 7：多 Agent 日报架构设计

本文档描述 `ai-intel-daily` 在阶段 7 中的多 Agent 架构设计。这个架构的核心目标是：未来可以替换模型供应商，而不是把项目写死为某一个模型或某一个 SDK。

阶段 7 只做设计和代码骨架，不调用真实 OpenAI API、真实 DeepSeek API、真实财经 API，或任何其他真实外部服务。本阶段也不提供确定性的买入、卖出、持有、重仓、目标价或交易建议。

## 为什么本项目后续需要多 Agent 架构

当前项目以本地优先方式生成 Markdown 日报。随着日报内容变复杂，多个职责需要清晰分离：

- 新闻整理需要关注来源、去重、主题归类和时间线。
- 市场信息整理可能涉及股票、行业、公司和风险，因此需要更严格的安全边界。
- 赛博投资委员会分析需要多角度辅助思考，但不能变成个性化投资建议。
- 事实检查需要区分事实、推断和不确定性。
- 金融安全检查需要拦截确定性交易语言。
- 最终编辑需要把内容整合成可读的 Markdown，同时保留引用、不确定性和免责声明。

多 Agent 设计可以让每个角色只承担窄职责、窄输入和明确 guardrails。这样未来实现时也更容易测试，因为每个 agent 都可以被单独 mock。

## Agents SDK 在本项目中负责哪一层

未来如果接入 Agents SDK，它应该只位于“编排层”。它负责协调 agent 步骤、工具访问、handoffs、guardrails 和最终报告组装。

Agents SDK 层不应该写死模型供应商。所有模型调用都应该通过模型供应商配置和客户端接口进入，这样项目未来可以切换到：

- OpenAI
- DeepSeek
- LiteLLM
- 本地模型
- mock 模型

阶段 7 不导入 `openai-agents`，也不调用任何真实模型供应商。本阶段第一个可用客户端是 mock client，只返回本地确定性文本。

## Agent 设计

### News Agent

- 职责：整理 AI 圈新闻，按主题归类，保留来源引用，并把信息不完整的内容标记为不确定。
- 输入：采集到的 RSS/Atom 条目、来源名称、URL、时间戳、短摘要。
- 输出：用于日报 Markdown 的结构化新闻摘要、来源列表、不确定性说明。
- 未来可用工具：RSS collector、网页抓取器、来源元数据解析器、去重器、主题分类器。
- 不允许做什么：编造来源、把未验证内容写成事实、在阶段 7 调用真实 API、输出交易结论。

### Market Agent

- 职责：整理 AI 相关市场信息、行业主题、业务指标观察点和风险因素。
- 输入：本地或未来的市场研究快照、行业列表、公司新闻、风险分类、模拟数据说明。
- 输出：明确标注为“研究辅助”的市场信息整理。
- 未来可用工具：市场快照读取器、财报日历读取器、公开文件读取器、行业分类辅助工具。
- 不允许做什么：阶段 7 连接真实财经 API，输出买入、卖出、持有、重仓、目标价、保证收益等语言，或使用非公开信息。

### Cyber Committee Agent

- 职责：复用现有赛博投资委员会风格做辅助分析，对比多个视角，列出分歧和后续观察问题。
- 输入：市场背景、公司或行业事实、风险列表、不确定性说明、金融安全政策。
- 输出：包含事实基础、角色观点、主要分歧、不确定性和后续观察指标的委员会式研究辅助分析。
- 未来可用工具：本地委员会角色模板、情景分析清单、风险分类辅助工具。
- 不允许做什么：提供个性化或确定性投资建议、选择“赢家”作为推荐、给出交易动作、弱化免责声明。

### Fact Check Agent

- 职责：检查来源覆盖、引用完整性，区分事实和推断，并标记缺少支持或不确定的陈述。
- 输入：News Agent、Market Agent、Cyber Committee Agent 的输出，以及来源元数据。
- 输出：事实检查说明、引用缺口、缺少支持的声明提醒、不确定性标签。
- 未来可用工具：引用检查器、来源 URL 校验器、引文检查器、去重器。
- 不允许做什么：把缺少支持的内容静默改写为事实、编造引用、跳过不确定性标记。

### Financial Safety Agent

- 职责：执行股票报告安全边界，检查免责声明，拦截确定性投资建议，确保报告只作为研究辅助。
- 输入：市场笔记、委员会分析、Markdown 草稿、金融安全政策。
- 输出：安全检查结果、被拦截短语、必须补充的免责声明说明。
- 未来可用工具：政策检查器、短语扫描器、报告章节校验器。
- 不允许做什么：批准确定性的买入、卖出、持有、重仓、保证收益、目标价或交易指令语言。

### Editor Agent

- 职责：把已通过检查的内容整合为最终 Markdown 日报，保留引用和免责声明，并保持事实、推断、不确定性分区清晰。
- 输入：前序 agent 的已检查输出，以及最终安全检查结果。
- 输出：最终 Markdown 报告草稿。
- 未来可用工具：Markdown 模板渲染器、章节排序辅助工具、风格检查器。
- 不允许做什么：移除安全免责声明、把不确定性改成确定结论、编造来源、添加交易建议。

## Agent 协作流程

1. News Agent 从允许的本地输入中整理 AI 新闻。
2. Market Agent 从允许的本地或模拟输入中整理 AI 相关市场研究背景。
3. Cyber Committee Agent 基于市场背景生成多视角辅助分析。
4. Fact Check Agent 检查来源覆盖、引用、缺少支持的声明和不确定性标签。
5. Financial Safety Agent 检查所有股票相关语言和必须出现的免责声明。
6. Editor Agent 只有在 workflow 中明确包含 fact check 和 financial safety check 后，才整合最终 Markdown 报告。

## 未来可以使用 tools 的地方

- News Agent 可以使用 RSS/Atom 采集工具。
- Fact Check Agent 可以使用来源校验和引用检查工具。
- Market Agent 可以使用本地市场快照读取工具。
- 在后续明确批准后，可以增加公开文件、财报电话会或财报日历读取工具。
- Financial Safety Agent 可以使用金融政策扫描工具。
- Editor Agent 可以使用 Markdown 模板渲染工具。

阶段 7 不接入任何真实外部 API 工具。

## 未来可以使用 handoffs 的地方

- News Agent 在发现缺少来源的声明时，handoff 给 Fact Check Agent。
- Market Agent 在发现股票、ticker、市值、估值或交易敏感语言时，handoff 给 Financial Safety Agent。
- Cyber Committee Agent 在角色观点依赖不确定证据时，handoff 给 Fact Check Agent。
- Fact Check Agent 带着必要修改和不确定性标签 handoff 回 Editor Agent。
- Financial Safety Agent 只有在不安全语言被删除或明确拦截后，才 handoff 回 Editor Agent。

## 必须加 guardrails 的地方

当报告包含市场或股票相关内容时，最终 Markdown 输出前必须执行 guardrails。

必须包含的 guardrails：

- 不允许确定性的买入、卖出、持有、重仓或交易建议。
- 不允许保证收益、保证盈利或确定性价格结果。
- 不允许目标价指令或个性化交易动作。
- 不允许使用未披露的非公开信息。
- 必须保留免责声明：报告仅用于研究辅助，不构成投资建议，不提供买入、卖出或持有建议。
- 事实、推断和不确定性必须保持可见分离。
- 真实 API 调用必须保持关闭，直到后续阶段明确新增并测试供应商适配器。

## 阶段 7 暂不做什么

- 不导入或配置 `openai-agents`。
- 不调用真实 OpenAI API。
- 不调用真实 DeepSeek API。
- 不调用 LiteLLM 或本地模型服务。
- 不连接真实财经 API。
- 不创建定时任务、n8n workflow、数据库、向量库、外部缓存或外部存储。
- 不把 API key 或凭据写入仓库。

只有在 mock 接口、guardrails 和测试稳定之后，后续阶段才可以增加真实模型供应商适配器。
